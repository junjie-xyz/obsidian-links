## Project Instructions

- Do not use real local paths, vault names, note names, personal data, or user-specific content in code, tests, README, UI copy, placeholders, screenshots, or examples.
- Use fictional examples only, such as `Work`, `DemoVault`, `Inbox/Test.md`, `Example Note`, and `Example query`.
- If a real user path or note title appears during debugging, treat it as private context and replace it before committing or publishing.
- When changing browser assets that can be cached (`assets/app.css`, `assets/pages.js`, or `assets/obsidian-links.js`), bump the `v=YYYYMMDDHHMMSS` cache-busting query in every HTML entrypoint and in the `assets/pages.js` import of `assets/obsidian-links.js`.
